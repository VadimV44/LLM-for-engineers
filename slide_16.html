Итеративность: как работать с LLM через диалог

📋 Ключевые тезисы
  • ✔ Первый промпт редко идеален — модель не читает мысли, нужно уточнять
  • ✔ Диалог = итерации — каждый ответ — черновик для доработки
  • ✔ Типы уточнений: добавь цифры, переформулируй вывод, добавь таблицу, исключи общие фразы
  • ✔ Техника «5 почему» — если ответ поверхностный, спрашивайте «почему?» 5 раз
  • ✔ Сохранение контекста — в рамках диалога модель помнит историю

✅ Примеры промптов
💬 Итеративное уточнение
Инженер: «Составь ТЗ на сварочный робототехнический комплекс для сварки кузовов»
LLM: [Модель выдаёт общий документ]
Инженер: «Добавь конкретные требования к точности: ±0.1 мм. Укажи производительность: 60 деталей/час»
LLM: [Модель дорабатывает документ]
Инженер: «Добавь раздел 'Требования к интеграции с MES-системой'. Укажи протоколы: OPC UA, MQTT»
LLM: [Модель добавляет раздел]
Инженер: «Исключи общие фразы из раздела 'Безопасность'. Оставь только конкретные требования по ГОСТ Р ИСО 10218-1»
LLM: ✅ Готовый документ
🔍 Техника «5 почему»
LLM: «Почему возник дефект выплеск при сварке?»
Инженер: Высокий ток сварки.
LLM: «Почему ток был высоким?»
Инженер: Неверно настроены параметры.
LLM: «Почему параметры неверны?»
Инженер: Отсутствует стандарт настройки.
LLM: «Почему нет стандарта?»
Инженер: Не разработан техпроцесс.
LLM: «Почему не разработан техпроцесс?»
Инженер: Нет данных по материалам.
LLM: Корневая причина: отсутствие базы данных по материалам.
Мероприятие: разработать базу данных.
💡 Практическая рекомендация
Сохраняйте удачные диалоги как шаблоны для будущих задач.
